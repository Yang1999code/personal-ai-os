# 维护目录说明

`维护/` 收纳不适合放在根目录的后台文件：完整台账、更新记录、设计说明、后续计划和给其他 AI 的整理提示词。

## 文件说明

| 文件 | 给谁看 | 用途 |
| --- | --- | --- |
| `全量索引.md` | Agent（智能体）/维护者 | 完整 source/wiki 台账 |
| `当前上下文交接.md` | Agent | 当前重点主题、写入原则和持续追加规则 |
| `经验库设计说明.md` | 人/维护者 | 经验库的设计背景、分层和使用说明 |
| `log.md` | Agent/维护者 | 每次新增、移动、重命名和重要更新记录 |
| `roadmap.md` | 人/维护者 | 后续改进计划，大改动先写这里 |
| `给其他AI的整理提示词.md` | Agent | 让其他 AI 按本库规则整理新资料 |

## 白话总结

`维护/` 是后台办公室。平时查资料不用先进这里，只有 Agent 维护目录、追历史记录、看后续计划时才进来。
