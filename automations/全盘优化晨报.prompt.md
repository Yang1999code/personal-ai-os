# 全盘优化晨报 Prompt（提示词）

每天运行一次“全盘优化晨报”只读审查。目标：检查前一个自然日内 Codex 和 Claude Code 的会话/记录，判断有没有内容值得沉淀到 `~/Desktop/my知识库`，有没有内容值得改进 `~/Desktop/myloop` 或 `~/Desktop/myskill`。只输出等待用户确认的建议和计划，不要自动修改任何文件。

## 展示要求

- 输出要像“晨报”，不要像审计报告。
- 尽量白话、短句、少堆路径，让用户早上能轻松阅读。
- 必须使用 Mermaid（流程图）画一张“昨天经验 -> 三层判断 -> 待确认建议”的总图。
- 少列绝对路径；必要时只列对应文件夹和目标页面。
- 必须把建议分成三块，并且每块先用一句白话解释这一层是什么：
  1. `loop（循环 / 方法论）`：对应 `~/Desktop/myloop`。这类建议是“以后遇到复杂任务时，AI 应该怎么一步步推进、检查、复盘”。
  2. `skill（技能 / 可复用流程）`：对应 `~/Desktop/myskill`。这类建议是“以后用户一句话就能叫出来的固定技能入口或流程”。
  3. `md/wiki（Markdown 文档 / 经验库）`：对应 `~/Desktop/my知识库`。这类建议是“这次学到的经验，下次不要再从零解释”。
- 每条建议必须明确：建议内容、为什么、建议放哪里、风险、是否等待用户确认。
- 不要直接改 `my知识库`、`myloop`、`myskill`，只给建议；除非用户后续明确确认。

## 工作步骤

1. 确定审查日期范围：以前一个自然日为主，使用本机时区。
2. 只读取本机权限允许且明显相关的材料。优先检查：`~/.codex/sessions`、`~/.codex/memories`、`~/.claude`、`~/Desktop/my知识库`、`~/Desktop/myloop`、`~/Desktop/myskill`。如果某个路径不存在或读不到，明确说明，不要假装读过。
3. 在 Codex 和 Claude Code 的前一天会话中，筛选真正有复用价值的内容：新流程、反复踩坑、工具配置、排查清单、Loop 规则、Skill 触发入口、知识库页面需要补充的结论。
4. 对每个候选项判断应该进入哪一层：loop、skill、md/wiki、AGENTS.md、CLAUDE.md，或者不建议沉淀。
5. 结合昨天真实工作里的痛点，指出“哪些本来可以自动化、模板化、检查清单化或沉淀成知识库”。
6. 轻量查看当日 AI 使用相关的新技术、工具、报告或行业动态，只挑 1-2 个和用户个人 AI 操作系统相关的灵感点。不要写新闻长文；只写“这对我今天有什么启发”。如果无法联网或没有可靠来源，就明确说明，并基于本地经验给灵感。

## 输出格式

```md
# 全盘优化晨报

## 一句话结论

用 2-3 句话讲清昨天最值得沉淀什么、今天建议用户确认什么。

## 今日总图

用 Mermaid（流程图）展示：昨天记录 -> loop / skill / md 判断 -> 等待用户确认。

## 昨天真正的痛点

用白话列 2-4 条：昨天哪些地方重复、卡住、可以自动化、可以流程化、可以沉淀。

## 1. loop（循环 / 方法论）建议

先用一句话解释：loop 类似方法论。
再列建议。每条包含：建议内容、为什么、建议放哪里、风险、是否等待用户确认。

## 2. skill（技能 / 可复用流程）建议

先用一句话解释：skill 类似技能流程。
再列建议。每条包含：建议内容、为什么、建议放哪里、风险、是否等待用户确认。

## 3. md/wiki（Markdown 文档 / 经验库）建议

先用一句话解释：md/wiki 类似记录经验库。
再列建议。每条包含：建议内容、为什么、建议放哪里、风险、是否等待用户确认。

## 不建议沉淀的东西

用白话说明哪些只是流水账或一次性操作。

## 今日 AI 灵感

结合当日 AI 使用的新技术、工具、报告或本地经验，给 1-2 条和用户个人 AI 操作系统相关的启发。必须落到“今天可以怎么用”。

## 今日 15 分钟行动

不要写泛泛的飞轮口号。必须结合昨天真实痛点和今日 AI 灵感，给 3-5 个具体动作，每个动作最好能在 3-5 分钟完成。

## 需要你确认

只问 1-3 个最关键问题，方便用户回复“确认第几条”。

## 白话总结

用最直白的话收尾。
```

## 白话总结

这个 prompt（提示词）让 AI 每天早上只读复盘：昨天哪些经验值得留下，应该进 loop、skill 还是 md/wiki。它不自动改文件，只给你确认用的建议。
