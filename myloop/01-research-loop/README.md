# 01 Research Loop

这是查找/需求 Loop（循环）。

适合目标还不够清楚时使用：先读项目、查本地知识库、查内部知识库、必要时上网，最后输出需求、证据、Done（完成）标准和需要人确认的问题。

## 什么时候用

- 用户只说了一个方向，还没说清楚要做什么。
- 需要先查资料、读规范、读历史记录。
- 需要判断“这个需求到底是不是合理”。
- 需要先产出 requirements（需求说明）和 done（完成标准），再交给创建/实现 Loop。

## 文件夹

| 文件夹 | 用途 |
| --- | --- |
| `00-intake/` | 用户原始问题、目标、限制 |
| `01-local-knowledge/` | 本地项目资料和 `$HOME/Desktop/my知识库` 摘要 |
| `02-baidu-ku/` | 内部知识库资料记录 |
| `03-web-research/` | 网络搜索记录 |
| `04-evidence/` | 证据表、来源、可信度 |
| `05-requirements/` | 需求说明、Done 标准 |
| `06-open-questions/` | 需要问人的问题 |
| `07-outputs/` | 最终调研报告 |
| `99-skills/` | 本 Loop 推荐使用的 skill（技能） |

## 推荐 skill

见 `99-skills/recommended-skills.md`。

## 白话总结

这个 Loop 不急着干活，先把问题查清楚、证据摆出来、目标定明白。
