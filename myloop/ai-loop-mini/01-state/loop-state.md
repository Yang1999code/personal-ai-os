# Loop State

## 当前目标

搭建一个轻脚手架、信任 AI 主动推进的小型 Loop（循环）工作区。

## Done 条件

- AI 能读取 `loop.md` 后自主跑一轮
- 每轮能更新状态、历史、输出和优化记录
- 复杂任务时能采用 maker-checker（生产者-检查者）思路
- 只在不可逆、外部副作用、敏感信息等硬边界前停下问人

## 运行模式

自由模式

可选：

- 自由模式：AI 自主判断继续、停止和下一步
- 协作模式：AI 每轮建议继续/停止，人确认
- 自动模式：按最大轮次、Done 条件和停止条件自动推进

## AI 可自主决定

- 本轮最值得做什么
- 是否需要读取额外材料
- 是否需要创建辅助记录
- 是否需要用 tester（测试者）视角复核
- 下一轮应该如何优化 Loop（循环）
- 是否需要围绕 `09-target/target.md` 中的目标地址或目标文件夹展开工作
- 是否需要更新 `09-target/verification.md` 中的验证方式

## 需要问人的情况

- 删除、覆盖、批量移动重要文件
- 安装依赖
- 提交、推送、部署
- 访问敏感信息
- 对外部系统产生真实影响
