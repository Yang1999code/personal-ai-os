# Verification

这里记录当前目标的验证方式。

验证方式不需要一开始写死。AI 可以根据目标类型主动补充、修改和优化。

## 验证取向

默认相信 AI 判断，但要求 AI 留下证据。

如果任务适合硬验证，优先使用硬验证；如果任务偏探索、研究、整理，可以使用软验证。

## 硬验证

适合代码、构建、测试、文件产物等任务。

示例：

```text
- 运行 npm test
- 运行 npm run lint
- 检查目标文件是否存在
- 检查 git diff 是否只包含相关文件
```

## 软验证

适合研究、文档、产品想法、结构设计等任务。

示例：

```text
- 是否覆盖了关键问题
- 是否有清楚的下一步
- 是否有证据或来源支撑
- Tester（测试者）是否认为结论可信
```
