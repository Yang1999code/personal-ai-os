# 04 Meta Loop

这是元loop（元学习 Loop）。

它不是直接做任务，而是回看一次完整任务中“人是怎么带任务的”，学习人的提问方式、纠偏方式、收窄方式、分层方式和确认方式，再把这些模式整理成可反哺其他 Loop（循环）的规则。

## 什么时候用

- 一个任务已经走完，想复盘“这个人是怎么想的”。
- 想从一次完整会话里提炼人的处理方式，而不是只总结任务结果。
- 想把“人的工作法”反哺到 `research/create/test` 这些 Loop。
- 想校正 AI 对人的理解，避免只靠一次聊天就过拟合。

## 核心结构

这是一个“总控 + 四个子循环”的学习型 Loop：

1. `01-replay-loop/`
   只做任务回放，回答“发生了什么”。
2. `02-intent-inference-loop/`
   推断人的意图，回答“为什么这样问、这样改、这样卡边界”。
3. `03-rule-abstraction-loop/`
   抽象成候选规则，回答“哪些模式值得进入 Loop”。
4. `04-alignment-loop/`
   多和人对话，确认“AI 学到的是不是用户真正想要的自己”。

## 文件夹

| 文件夹 | 用途 |
| --- | --- |
| `00-intake/` | 本次要学习的会话范围、任务边界、目标说明 |
| `01-replay-loop/` | 回放完整任务流程 |
| `02-intent-inference-loop/` | 推断人的意图和处理偏好 |
| `03-rule-abstraction-loop/` | 抽象候选规则和适用边界 |
| `04-alignment-loop/` | 和人对齐，确认哪些规则真正成立 |
| `05-synthesis/` | 汇总本轮学习结论 |
| `06-feed-back/` | 反哺到其他 Loop 的建议 |
| `07-outputs/` | 最终报告、稳定规则、待观察规则 |
| `99-skills/` | 可参考的 skill（技能）、资料和调用建议 |

## 白话总结

这个 Loop 学的不是任务本身，而是“你这个人是怎么带任务的”。它像一个观察员、复盘员和教练，把人的工作法抽出来，再反哺给其他 Loop。
