# 修改记录

这是本 Loop（循环）的中文修改总账。

以后每次修改本模板或由本模板生成的任务专用 Loop，都要先后记录：

- 改前：修改前是什么状态，涉及哪些文件或规则。
- 改后：修改后变成什么，新增/删除/移动/重写了什么。
- 原因：为什么要改，解决什么问题。
- 隐患：可能带来的副作用、兼容风险、后续维护成本。
- git 联动：对应本地 git（版本控制）提交号；如果还没提交，先写“待提交”，提交后补充。

## 本地 git 规则

- 本模板自身使用本地 git 记录模板变化。
- 每个真实任务生成的任务专用 Loop 也必须有自己的独立本地 git。
- 任务专用 Loop 初始化完成后，先提交一次 `Initial task loop`，用于固定起点。
- 任务专用 Loop 的 git 只记录该任务 Loop 自己的文件，不替目标项目做提交、推送或回滚。
- 需要修改目标项目代码时，必须先读取目标项目自己的 git 状态，再按目标项目规则处理。

## 记录模板

```text
## YYYY-MM-DD HH:mm - 标题

- 改前：
- 改后：
- 原因：
- 隐患：
- git 联动：
```
